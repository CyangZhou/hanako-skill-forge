# 02-需求卡协议.md

## 任务卡

```md
# 技能需求卡
- 技能名称：
- 当前模式：新建 / 重构 / 维护 / 扩展 / 自愈
- 目标用户：
- 触发词：
- 目标结果：
- 要解决的问题：
- 核心输入：
- 核心输出：
- 文档主来源：
- 可选脚本动作：
- 必须保留：
- 必须删除：
- 表达要求：只保留功能必要语句 / 删除花哨无用语句
- 改造链要求：维护 / 扩展 / 自愈作为后续修改路径保留
- 本轮最小交付物：
- 证据要求：
- 验收标准：
```

## 填写规则

- 技能名、输入、输出必须先定；
- 先写目标结果，再写动作；
- 文档主来源必须写清；
- 脚本只能写“可选脚本动作”，不能默认先上脚本；
- 保留项、删除项、改造链要求必须分开；
- 明确哪些语句是功能必要，哪些应删除；
- 信息不全时，先补任务卡再走路由。
