# 01-总控路由.md

## 路由判断

### A. 自愈
- 当前技能存在断链、失效、冲突、漂移；
- 需要先修复，再继续演进。

继续读取：`modules/recovery/15-自愈协议.md`

### B. 维护
- 当前核心问题是结构收口、主来源一致、引用一致；
- 需要先做健康检查和同步更新。

继续读取：`modules/maintenance/13-维护协议.md`

### C. 扩展
- 当前架构已成立，但能力不够；
- 需要补新模块、新模板或新脚本辅助。

继续读取：`modules/extension/14-扩展协议.md`

### D. 新建 / 封装 / 重构
- 还没有成型技能；或
- 已有技能但要重新架构；或
- 已有流程、提示词、脚本要收成规范技能。

继续读取顺序：
1. `modules/research/11-对标与能力注入协议.md`
2. `modules/build/12-架构与落地协议.md`
3. `modules/review/16-验收与发布协议.md`

## 输出格式

```md
- 当前路由：自愈 / 维护 / 扩展 / 新建 / 封装 / 重构
- 下一步：
- 本轮最小闭环：
- 需要的证据：
- 暂不处理：
```
