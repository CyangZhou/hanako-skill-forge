# 18-触发评测样例集.md

用途：用于检查 `hanako-skill-forge` 的描述符是否在该触发时触发、不该触发时不触发。

## should_trigger

1. 帮我新建一个 hanako 技能，用来把零散 SOP 收成可维护技能。
2. 这个旧技能越改越乱，帮我重构成清楚的模块结构。
3. 给这个技能补维护、扩展、自愈三条后续改造链。
4. 我想统一几个技能的入口、模板和验收口径。
5. 这个技能已经有历史残留和断链，帮我清理并收口。
6. 把这套提示词正式做成技能资产，不要再散着放。
7. 我想把技能改成文档主导、脚本辅助的结构。
8. 帮我给技能建立状态模板、检查点和交付摘要。
9. 这个技能不是坏了，是长期维护成本太高，帮我工程化。
10. 帮我做一个技能元技能，用来维护别的技能。

## should_not_trigger

1. 把这句描述改顺一点。
2. 帮我修一下 Python 报错。
3. 这个网页帮我点一下按钮。
4. 我有个 AI 工作流想法，先陪我聊聊是否值得做。
5. 帮我查一下 OpenHanako 最新版本。
6. 给这份 README 润色一下英文。
7. 只改一个文件里的标题格式。
8. 这个函数返回值不对，帮我定位 bug。
9. 帮我测试这个技能描述的触发准确率。
10. 帮我做一套 skill benchmark，对比改版前后命中率。

## 判定说明

- `should_trigger` 主要覆盖：新建、重构、收口、治理、维护链、扩展链、自愈链、验收链；
- `should_not_trigger` 主要覆盖：文案微调、小 bug、网页执行、泛化工作流讨论、纯评测任务；
- 其中“技能评测、benchmark、触发优化”应优先路由到 `skill-creator`，而不是本技能。
