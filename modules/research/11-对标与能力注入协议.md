# 11-对标与能力注入协议.md

## 对标顺序

1. 先查本地已有技能；
2. 再查当前仓库里可复用模块；
3. 再查开源优秀框架；
4. 最后再决定新增什么。

## 对标输出

- 保留：当前结构里已经够好的部分；
- 注入：要从本地或开源吸收的新能力；
- 延后：有价值但这轮不该做的部分；
- 删除：会带来冗余、混乱、脚本膨胀的部分。

## 能注入什么

- 原则层能力：如规格先行、渐进展开；
- 模块层能力：如维护、扩展、自愈分支；
- 资源层能力：如引用巡检、术语巡检、最小验证脚本；
- 沉淀层能力：如通过验证后写入可复用清单。

## 规则

- 能复用就不重写；
- 能写成文档规则就不急着写脚本；
- 引入开源能力时，必须写明来源、作用、落点；
- 只吸收能力，不搬运废话；
- 对标后必须形成 `保留 / 注入 / 延后 / 删除` 四类清单。
